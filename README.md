# Tropical Sensor Dashboard

A Vercel-friendly static dashboard for viewing `sensor_data` temperature and humidity readings.

## 1) Run the SQL schema

Use `schema.sql` in your Postgres/Supabase SQL editor:

```sql
create table sensor_data (
  id bigint generated by default as identity primary key,
  temperature_c float,
  temperature_f float,
  humidity float,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
```

## 2) Deploy to Vercel

- Push this repo to GitHub.
- Import it in Vercel.
- Framework preset: **Other** (static site).
- No build command needed.

## 3) Dashboard behavior

- Supabase project URL and anon key are already configured in `app.js`.
- Continuous auto-refresh polling (5s to 60s slider)
- Threshold sliders for temperature and humidity alerts
- In-page alert banner + optional browser notification when thresholds are crossed (with notification cooldown)
- Manual refresh button is above the graph
- Medium-sized chart tuned for single-page viewing

The page fetches up to 50 latest rows from `sensor_data` and renders:
- current temperature (°C)
- current humidity (%)
- a line chart for both trends
